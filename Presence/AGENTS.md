# Presence iOS App Rules

## Architecture
- Use MVVM architecture only
- Separate Views, ViewModels, Models, and Services
- No business logic inside SwiftUI Views
- Use SwiftUI only
- Use modern iOS APIs
- Use @Observable instead of ObservableObject
- Use async/await for asynchronous work
- Use SwiftData for local persistence

## UI Rules
- Minimal Apple-style design
- Dark mode first
- Reusable components
- Use Swift Charts for analytics
- Use SF Symbols only

## Code Quality
- Keep files modular
- Avoid giant Views
- Prefer composition
- Add MARK comments

## Constraints
- Local-first app
- No backend initially
- No authentication
- Privacy-focused implementation

## App Goal
Presence is a behavioral analytics app that tracks:
- sleep
- screen time
- focus
- recovery trends
- behavioral patterns

## iOS 26 Design System
- Use modern iOS 26 design principles
- Use Liquid Glass subtly and intentionally
- Prefer clarity and readability over visual effects
- Use translucency mainly for cards, overlays, and navigation surfaces
- Avoid excessive blur and transparency
- Use layered depth carefully
- Use floating card layouts
- Use smooth motion and transitions
- Use immersive dark mode aesthetics
- Maintain strong visual hierarchy

## Animation Rules
- Use smooth spring animations
- Prefer subtle motion over flashy effects
- Keep interactions fluid and responsive
- Avoid distracting animations

## Layout Rules
- Use generous spacing
- Prefer modular card-based layouts
- Use clean typography
- Avoid cluttered dashboards
- Prioritize readability and hierarchy

## UX Philosophy
Presence should feel:
- calm
- premium
- intelligent
- futuristic
- analytical

The UI should resemble:
- Apple Health
- Apple Journal
- modern iOS system apps

Avoid:
- excessive glassmorphism
- neon colors
- childish wellness aesthetics
- over-animated interfaces
